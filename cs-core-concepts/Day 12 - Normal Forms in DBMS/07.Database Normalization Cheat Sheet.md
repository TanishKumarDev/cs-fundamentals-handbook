# **Day 12 ‚Äî Database Normalization Cheat Sheet**

---

## **1Ô∏è‚É£ First Normal Form (1NF)**

**Definition:**

* Each table column must hold **atomic (indivisible) values**.
* No repeating groups or arrays.

**Key Rule:**

* Every cell contains **one value only**.

**Example:**

| Student | Subjects      |
| ------- | ------------- |
| A       | Math, Science |

**1NF Table:**

| Student | Subject |
| ------- | ------- |
| A       | Math    |
| A       | Science |

**Purpose:** Remove multi-valued attributes.

---

## **2Ô∏è‚É£ Second Normal Form (2NF)**

**Definition:**

* Table must be in **1NF**.
* **No partial dependency**: no non-prime attribute depends on part of a candidate key.

**Example:**

| StudentID | CourseID | CourseName |
| --------- | -------- | ---------- |

* Candidate Key: {StudentID, CourseID}
* CourseName depends only on CourseID ‚Üí **partial dependency**

**2NF Tables:**

1. **Student_Course** ‚Üí {StudentID, CourseID}
2. **Course** ‚Üí {CourseID, CourseName}

**Purpose:** Eliminate partial dependencies.

---

## **3Ô∏è‚É£ Third Normal Form (3NF)**

**Definition:**

* Table must be in **2NF**.
* **No transitive dependency**: non-key attributes cannot depend on other non-key attributes.

**Example: Candidate Table**

| CAND_NO | CAND_NAME | CAND_STATE | CAND_COUNTRY |
| ------- | --------- | ---------- | ------------ |

* CAND_NO ‚Üí CAND_STATE ‚Üí CAND_COUNTRY (**transitive dependency**)

**3NF Tables:**

1. **Candidate** ‚Üí {CAND_NO, CAND_NAME, CAND_STATE}
2. **State_Country** ‚Üí {CAND_STATE, CAND_COUNTRY}

**Purpose:** Remove transitive dependencies & anomalies.

---

## **4Ô∏è‚É£ Boyce-Codd Normal Form (BCNF)**

**Definition:**

* Table must be in **3NF**.
* **Every determinant must be a candidate key**.

**Why needed:**

* 3NF may allow redundancy if a non-superkey determinant exists.

**Example:**

| Student | Teacher | Subject |
| ------- | ------- | ------- |

FDs:

* (Student, Teacher) ‚Üí Subject
* (Teacher ‚Üí Subject) ‚Üí violates BCNF because Teacher is not a key

**BCNF Tables:**

1. **Teacher_Subject** ‚Üí {Teacher, Subject}
2. **Student_Teacher** ‚Üí {Student, Teacher}

**Purpose:** Ensure all determinants are candidate keys, remove hidden redundancy.

---

## **5Ô∏è‚É£ Fourth Normal Form (4NF)**

**Definition:**

* Table must be in **BCNF**.
* **No multivalued dependencies (MVDs)** except on candidate keys.

**Multivalued Dependency:**

* One attribute determines multiple independent values of another attribute.
* Example: A course has multiple instructors and multiple textbook authors **independently**.

**Example Table:**

| Course | Instructor | Textbook_Author |
| ------ | ---------- | --------------- |

* Instructor and Author are independent ‚Üí violates 4NF

**4NF Tables:**

1. **Course_Instructor** ‚Üí {Course, Instructor}
2. **Course_Author** ‚Üí {Course, Textbook_Author}

**Purpose:** Remove redundancy from independent one-to-many relationships.

---

## **6Ô∏è‚É£ Fifth Normal Form (5NF / PJNF)**

**Definition:**

* Table must be in **4NF**.
* **No join dependencies** that can cause redundancy.
* Decomposition must be **lossless**.

**Example Table:**

| Subject | Class | Teacher |
| ------- | ----- | ------- |

* Multiple teachers can teach multiple subjects in multiple classes.

**Decomposition:**

1. R1 ‚Üí {Subject, Class}
2. R2 ‚Üí {Class, Teacher}
3. R3 ‚Üí {Subject, Teacher}

* Joining R1 ‚®ù R2 ‚®ù R3 ‚Üí original table **without loss**

**Purpose:** Eliminate redundancy from join dependencies; ensure lossless decomposition.

## ‚úÖ **Summary Table of Normal Forms**

| Normal Form | Requirement                                | Eliminates         | Example Focus                |
| ----------- | ------------------------------------------ | ------------------ | ---------------------------- |
| 1NF         | Atomic values                              | Multi-valued attr  | Subjects list ‚Üí single rows  |
| 2NF         | No partial dependency                      | Partial dependency | Candidate key split          |
| 3NF         | No transitive dependency                   | Transitive dep     | Candidate ‚Üí State ‚Üí Country  |
| BCNF        | Every determinant is a candidate key       | Hidden redundancy  | Teacher ‚Üí Subject            |
| 4NF         | No multivalued dependencies (MVDs)         | Independent MVDs   | Course ‚Üí Instructor & Author |
| 5NF / PJNF  | No join dependency, lossless decomposition | Join redundancy    | Subject-Class-Teacher        |

---

### üí° **Key Takeaways**

* Normalization improves **data integrity**, **reduces redundancy**, and **prevents anomalies**.
* As normal forms increase: 1NF ‚Üí 2NF ‚Üí 3NF ‚Üí BCNF ‚Üí 4NF ‚Üí 5NF ‚Üí **database becomes more structured but complex**.
* **BCNF** and above handle edge cases not covered by 3NF.
